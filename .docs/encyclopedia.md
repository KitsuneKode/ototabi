# Encyclopedia — Ototabi Concepts

## Core Domain

| Term            | Definition                                                                                                                          |
| --------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| **Room**        | A virtual studio. Has a unique join code (e.g. `abc-1234`). Created by a host. Contains chat, participants, and recording sessions. |
| **Host**        | The user who created the room. Controls recording start/stop.                                                                       |
| **Guest**       | A participant who joined via the room code. No account needed (Phase 2).                                                            |
| **Participant** | Any user present in a LiveKit room, host or guest. Tracked via `RoomParticipant`.                                                   |
| **Join Code**   | A short alphanumeric code (e.g. `abc-1234`) used to join a room. Auto-generated on room creation with collision retry.              |

## Recording

| Term                  | Definition                                                                                                                |
| --------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| **Recording Session** | A single recording event in a room. Status: `RECORDING` → `COMPLETED`. Contains multiple tracks.                          |
| **Track**             | One media stream from one participant. Types: `CAMERA`, `MICROPHONE`, `SCREENSHARE`. Each track is a separate file in S3. |
| **Chunk**             | A ~4-second slice of recorded media. Stored in IndexedDB + OPFS before upload.                                            |
| **SID**               | LiveKit track identifier. Used to correlate local recordings with LiveKit tracks.                                         |
| **Track SID**         | LiveKit `Track.Sid` — unique per track in a LiveKit room. Stored in DB to match uploads to participants.                  |
| **Sync Marker**       | A timestamp pair (local + server) recorded every 2s during recording. Used to align tracks in post-production.            |

## Upload

| Term                 | Definition                                                                                                            |
| -------------------- | --------------------------------------------------------------------------------------------------------------------- |
| **Multipart Upload** | S3 upload broken into parts. Enables resume after failure. Initiate → upload parts → complete.                        |
| **Upload ID**        | S3-generated identifier for a multipart upload session. Stored in IndexedDB for recovery.                             |
| **Presigned URL**    | Time-limited S3 URL generated by the server. Client PUTs chunk data directly to S3.                                   |
| **Upload Resume**    | On reconnect, `ListParts` retrieves already-uploaded parts → client skips them → continues from last incomplete part. |

## Storage

| Term          | Definition                                                                                                                                   |
| ------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| **IndexedDB** | Browser database. Stores chunks + upload session state via Dexie.js. Queryable but can be evicted by Chrome under memory pressure.           |
| **OPFS**      | Origin Private File System. Persistent file storage in the browser. Not evictable. Used as primary chunk storage with IndexedDB as fallback. |
| **MinIO**     | Self-hosted S3-compatible object storage. Runs in Docker on port 9000. Auto-creates `ototabi-recordings` bucket.                             |

## Post-production

| Term            | Definition                                                                                                                                                |
| --------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **FFmpeg.wasm** | WebAssembly build of FFmpeg. Runs entirely in the browser. Used for: merge tracks, re-encode (720p/1080p), trim clips, noise reduction, subtitle burn-in. |
| **Transcript**  | Word-level timestamped text of all spoken content in a session. Generated by Whisper after recording completes.                                           |
| **Chapter**     | A labeled time range in a recording. Auto-generated by LLM from transcript.                                                                               |
| **Show Notes**  | LLM-generated summary, key topics, and SEO titles from a transcript.                                                                                      |
| **Clip**        | A short video segment extracted from a recording. Can be vertical, horizontal, or square. With burned-in captions.                                        |

## Design System

| Term               | Definition                                                                                                                                               |
| ------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Retro Analog**   | Visual language inspired by vintage recording studio equipment, synthesizers, and mechanical interfaces.                                                 |
| **Chassis**        | Outer container (`AnalogCard`) with raised bevel + drop shadow. Feels like equipment housing.                                                            |
| **Inset / Panel**  | Recessed area (`AnalogInset`) with inner shadow. Used for displays, meters, data read-outs.                                                              |
| **LED**            | Indicator lights (`Led`, `LedInline`). Colors: red (recording/warn), green (ok/completed), amber (processing), off. Pulse animation during active state. |
| **btn-mechanical** | CSS utility for raised physical buttons with press-down behavior (`active:translate-y-[2px]` + shadow compression).                                      |
| **Noise Texture**  | Fullscreen SVG turbulence overlay for a mechanical aged-surface feel.                                                                                    |
| **Scanlines**      | CRT-style horizontal line overlay on video areas. Pure CSS via gradient background.                                                                      |

## Retro Analog v2 (Product Shell)

| Term               | Definition                                                                                                        |
| ------------------ | ----------------------------------------------------------------------------------------------------------------- |
| **ProductShell**   | Shared page frame: noise texture, `min-h-[100dvh]`, centered `max-w-6xl` content column.                          |
| **SiteHeader**     | Marketing/app chrome: brand block, primary nav, PWR/SYNC LEDs, mobile menu.                                       |
| **Signal accent**  | Single amber-orange (`--accent`) for active UI — never purple gradients or multi-accent palettes.                 |
| **Section rhythm** | Major sections use `py-20 md:py-28` and `border-t-2` separators — chapters, not stacked cards.                    |
| **Feature bento**  | Asymmetric `grid-flow-dense` layout; outer radius = inner radius + padding (concentric radii).                    |
| **Motion**         | Enter from `scale(0.96)` + opacity; exits faster than enters; `--ease-mechanical` for press, `--ease-out` for UI. |

Default product theme: **dark** (studio console). Light mode remains available via system preference.
